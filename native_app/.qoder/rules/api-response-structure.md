# API 响应结构规范

## 统一响应格式 (Result\<T\>)

后端所有接口统一返回 `Result<T>` 结构：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": { ... }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `int` | 业务状态码，200 表示成功 |
| `msg` | `String` | 提示信息 |
| `data` | `T?` | 业务数据，失败时为 null |

## 分页响应格式 (PageResult\<T\>)

分页接口的 `data` 字段为 `PageResult<T>` 结构：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "current": 1,
    "size": 10,
    "total": 100,
    "pages": 10,
    "records": [ ... ]
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `current` | `int` | 当前页码 |
| `size` | `int` | 每页条数 |
| `total` | `int` | 总记录数 |
| `pages` | `int` | 总页数 |
| `records` | `List<T>` | 当前页数据列表 |

## 错误码体系

### 通用错误码

| code | 含义 | HTTP Status |
|------|------|-------------|
| 200 | 操作成功 | 200 |
| 400 | 请求参数错误 | 400 |
| 401 | 未登录或登录已过期 | 401 |
| 403 | 无访问权限 | 403 |
| 404 | 资源不存在 | 404 |
| 409 | 资源冲突 | 409 |
| 500 | 服务器内部错误 | 500 |

### 模块级错误码

按模块分段编号，格式为 `{模块序号}xxxx`：

| 模块 | 范围 | 示例 |
|------|------|------|
| 用户模块 | 10001 ~ 19999 | 10001 用户名已存在, 10002 用户不存在 |
| 后续模块 | 递增 | `{模块序号}xxxx` |

## Flutter 端处理要求

- 网络层统一解析 `Result<T>`，根据 `code` 判断成功/失败
- `code == 200` 时提取 `data` 字段映射为 Model
- `code != 200` 时抛出业务异常，携带 `code` 和 `msg`
- 分页请求使用 `PageResult<T>` 解析 `data` 字段
- 401 响应自动跳转登录页或触发 token 刷新
