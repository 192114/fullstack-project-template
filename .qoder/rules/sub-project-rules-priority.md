# 子项目规则优先级

当在子项目目录（如 `backend/`、`native_app/`）下生成或修改代码时，必须遵循以下规则优先级：

## 优先级声明

1. **子项目 `.qoder/rules/` 下的规则具有最高优先级** — 它们是代码实现的单一可信源（single source of truth）
2. **子项目 `.qoder/skills/` 下的 skill 作为实现参考** — 生成代码时应参考相关 skill 的指导
3. **根级规则不得覆盖子项目规则** — 根级规则仅负责跨项目协调和元级约束

## 执行要求

- 在对子项目代码进行任何操作前，必须先读取该子项目 `.qoder/rules/` 目录下的所有规则文件
- 如果根级指令（如 Skill 编排）与子项目规则存在冲突，以子项目规则为准
- 遇到冲突时应暂停执行并向用户报告，而非自行决定

## 适用范围

此规则适用于所有根级 Skill 和跨子项目的编排任务，包括但不限于：
- `/feature-execute` 工作流
- 任何从根目录发起的代码生成操作
